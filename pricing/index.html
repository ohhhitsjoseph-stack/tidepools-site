Frequently asked questions

One sample = one pair of demultiplexed, gzipped FASTQ files (R1 + R2) from a single extraction or collection event. If you upload paired-end reads from one library, that counts as one sample. Multiplexed runs should be demultiplexed prior to upload. Single-end submissions are supported but paired-end is recommended for DADA2 merge quality.
Unused samples do not roll over between billing periods. If you have seasonal spikes, the Lab plan's 150-sample allocation accommodates most batch processing needs. Enterprise customers can negotiate flexible volume commitments.
Oceara supports amplicon-based metabarcoding for COI (Leray/mlCOI), 12S rRNA (MiFish-U/E), 16S rRNA (515F/806R), and 18S rRNA (V4/V9 regions) from Illumina paired-end sequencing (2×150 to 2×300 bp). Oxford Nanopore amplicon data is supported in beta. Primer trimming parameters are auto-detected or user-configurable. Additional markers are being added based on demand.
Researcher and higher plans run three independent classification algorithms in parallel — BLAST+ sequence similarity against NCBI nt, Naive Bayes probabilistic classification (trained per-marker), and BOLD barcode database matching for COI. When methods produce conflicting assignments, the consensus engine evaluates percent identity, bootstrap support, and lowest common ancestor across all three and assigns a composite confidence score (0–100). This multi-method approach reduces false positives that any single classifier would propagate.

All data is transmitted over TLS 1.3 and encrypted at rest (AES-256). FASTQ files are processed in ephemeral containerized environments on Modal that are destroyed after pipeline completion — no sequence data persists on compute nodes. Results are stored in tenant-isolated Postgres partitions with row-level security policies. Sequence data is purged after your configured retention window. SOC 2 compliance is on the roadmap.
