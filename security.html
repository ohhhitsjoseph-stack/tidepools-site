Security & Compliance Features

Data Encryption

All eDNA data is encrypted end-to-end using AES-256 encryption standards. Data in transit is protected with TLS 1.3 encryption, ensuring your sequences and biodiversity results remain confidential throughout processing and storage.

Access Controls

Role-based access control (RBAC) ensures only authorized team members can view or modify data. Single Sign-On (SSO) integration with enterprise identity providers and multi-factor authentication (2FA) provide additional security layers for your laboratory.

Role-based access SSO integration Multi-factor auth

Compliance Certifications

Protecting your research data

How We Handle Your eDNA Data

01

Secure Data Ingestion

Your FASTQ files are uploaded through encrypted HTTPS connections and immediately stored in encrypted AWS S3 buckets with access logging enabled. All uploads are cryptographically verified to prevent tampering.

02

Confidential Processing

Analysis computations run in isolated, time-limited containers on AWS. Intermediate data is never stored unencrypted. After processing, temporary working data is automatically purged from compute instances within 24 hours.

03

Secure Storage & Access

Processed results remain encrypted at rest and in transit. Access to data requires authentication through your organization's SSO provider. All data access is logged and can be audited through our secure dashboard.
